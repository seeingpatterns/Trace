# Data Sources

## Canonical Source

**`films_embedded.json` (project root)** is the single source of truth for all film data.

- Generated by `embed_films.py` (Google Gemini API + UMAP + K-Means)
- Read by the backend at startup: `server.js:64` via `readFileSync()`
- Contains 106 films with embedding coordinates and cluster assignments
- To update film data, edit this file (or re-run the embedding pipeline)

## Derived Copy

**`public/films_embedded.json`** is a copy of the canonical file, served to the frontend.

- **DO NOT edit this file directly.** Changes will be overwritten.
- Synced from root via: `npm run sync:films`
- Automatically synced before builds via: `npm run prebuild`
- Vite serves this file at `/films_embedded.json` (dev server and production build)

## Frontend Data Loading (`src/data.js`)

`loadFilms()` tries three sources in order:

| Priority | Source | Films | Coordinates | Status/Review |
|----------|--------|-------|-------------|---------------|
| 1st | `GET /api/films` | 106 | From canonical JSON | From DB (reviews table) |
| 2nd | `GET /films_embedded.json` | 106 | From canonical JSON | All `unwatched`, no reviews |
| 3rd | `generateDemoData()` | 59 | Random (not real embeddings) | All `unwatched`, no reviews |

## Runtime Film Shape

The frontend UI depends on this exact shape from `loadFilms()`:

```
Film {
  title: string          // Korean title — card display, search
  title_en: string       // English title — DB key, API calls
  year: number           // Release year — DNA card decade distribution
  director: string       // Director — card display
  recommender: string    // Threads ID(s), "/" separated — treasure hunt, DNA card
  note: string           // Recommender's one-line comment — card display
  description: string    // Film description — card display, keyword extraction
  x: number              // Embedding X coordinate — constellation position
  y: number              // Embedding Y coordinate — constellation position
  cluster: number        // Cluster ID (0-6) — color, constellation lines, legend
  status: string         // "unwatched"|"watching"|"watched" — brightness/saturation
  review: string|null    // Reflection content — card display
}
```

**Notes:**
- Fields `title` through `cluster` (10 fields) come from the canonical JSON
- Fields `status` and `review` are added by `loadFilms()`:
  - API mode: from DB `reviews` table (`status` field + `content` renamed to `review`)
  - Fallback/demo mode: defaults (`"unwatched"`, `null`)
- The `recommender` field contains "/"-separated Threads handles for multi-recommender films
  - Example: `"chris_chang_arong / _seola_kim"`
  - Parsed at runtime by splitting on `/` with whitespace trimming

## Environments

| Environment | Frontend reads | Backend reads |
|-------------|---------------|---------------|
| Dev (`npm run dev`) | `public/films_embedded.json` via Vite dev server | `films_embedded.json` (root) via `readFileSync` |
| Build (`npm run build`) | `public/films_embedded.json` copied to `dist/` | N/A |
| Production (Vercel + Docker) | `dist/films_embedded.json` | `films_embedded.json` (root) in Docker container |

## Sync Rule

Before any build or deployment, run `npm run sync:films` to ensure `public/films_embedded.json`
matches the canonical root file. The `prebuild` hook does this automatically for `npm run build`.
